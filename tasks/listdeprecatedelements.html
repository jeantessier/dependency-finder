ListDeprecatedElements

Description

List of package names, class names, and feature names that are deprecated, one per line.

You can pass this output to deprecationreporter to get a list of areas in your code that depend on deprecated APIs.

Parameters

Attribute Description Required
destfile where to write the output. If the file does not exists, it is created. If it already exists, its content is overwritten. Yes

Nested elements

path

Use nested path elements to specify file names of JAR files, Zip files, and .class files for the tool to read. You can also give directory names, the tool will search it for .class files and read them. It is a path-like structures.

Examples

To scan the current directory and its subdirectories for .class files to analyze and save the resulting report to df.deprecated.txt:

  <listdeprecatedelements destfile="df.deprecated.txt">
    <path>
      <pathelement location="."/>
    </path>
  </listdeprecatedelements>

To scan DependencyFinder.jar and rt.jar for .class files to analyze and save the resulting report to df.deprecated.txt:

  <listdeprecatedelements destfile="df.deprecated.txt">
    <path>
      <pathelement location="DependencyFinder.jar"/>
      <pathelement location="rt.jar"/>
    </path>
  </listdeprecatedelements>
