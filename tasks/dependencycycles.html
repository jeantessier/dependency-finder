DependencyCycles

Description

Looks for cycles in a dependency graph starting from a given starting point.

This tool takes a dependency graph generated by DependencyExtractor and follows a trail of dependencies to look for cycles. The result is a list of cycles that it found in the graph. Each cycle lists the dependencies that make it in their proper order.

This tool can read multiple graphs. It will merge them together before it starts looking for cycles.

You can limit the number of dependencies on any cycles with maximumcyclelength. By default, the tool will go as far as possible looking for cycles.

With the xml attribute, it converts the cycles into an XML document that follows the cycles DTD.

Parameters

Attribute Description Required
srcfile filename of the XML document containing the dependency graph. Yes, unless nested <src> elements are present.
validate validate the input XML against its DTD. No; defaults to false.
destfile where to write the output. If the file does not exists, it is created. If it already exists, its content is overwritten. Yes
startincludes the name of a package, class, or feature must match the regular expressions for it to be included in the starting point. No
startexcludes the name of a package, class, or feature must NOT match the regular expressions for it to be included in the starting point. No
packagestartincludes the name of a package must match the regular expressions for it to be included in the starting point. No
packagestartexcludes the name of a package must NOT match the regular expressions for it to be included in the starting point. No
classstartincludes the name of a class must match the regular expressions for it to be included in the starting point. No
classstartexcludes the name of a class must NOT match the regular expressions for it to be included in the starting point. No
featurestartincludes the name of a feature must match the regular expressions for it to be included in the starting point. No
featurestartexcludes the name of a feature must NOT match the regular expressions for it to be included in the starting point. No
startincludeslist the name of a package, class, or feature must match an entry in the named files for it to be included in the starting point. No
startexcludeslist the name of a package, class, or feature must NOT match any of the entries in the named files for it to be included in the starting point. No
xml writes the dependency graph as an XML document with the dependencies DTD. No; defaults to false.
encoding encoding used for the XML output document. Only useful with xml. No
dtdprefix prefix of the DTD URL for the XML output document. Only useful with xml. No; defaults to "https://depfind.sourceforge.io/dtd".
indenttext used for each indentation level in textual and XML output. No; defaults to "    " (4 spaces).
maximumcyclelength maximum distance to travel by following outbound dependencies to find cycles. If the graph is big, the tool might spend a lot of time following very long chains of dependencies, looking for cycles. Use this attribute to restrict it to something manageable. No

Nested elements

src

<dependencycycles>'s srcfile attribute is a path-like structures and can also be set via a nested <src> element.

startincludeslist

<dependencycycles>'s startincludeslist attribute is a path-like structures and can also be set via a nested <startincludeslist> element.

startexcludeslist

<dependencycycles>'s startexcludeslist attribute is a path-like structures and can also be set via a nested <startexcludeslist> element.

Examples

To analyze the graph in df.xml and find all cycles and save them to de.cycles.xml:

  <dependencycycles srcfile="df.xml"
                    destfile="de.cycles.xml"/>

To analyze the graph in df.xml and find cycles that involve Classfile and its methods, of length at most 10:

  <dependencycycles srcfile="df.xml"
                    destfile="classfile.cycles.xml"
                    startincludes="/Classfile/"
                    maximumcyclelenth="10"/>

To analyze the graph in df.xml and find cycles between packages only, and convert them to HTML:

  <dependencyreporter srcfile="df.xml"
                      destfile="p2p.df.xml"
                      p2p="yes"
                      xml="yes"/>
  <dependencycycles srcfile="p2p.df.xml"
                    destfile="p2p.cycles.xml"
                    xml="yes"/>
  <xslt style="${dependencyfinder.home}/etc/CyclesToHTML.xsl"
        in="p2p.cycles.xml"
        out="p2p.cycles.html"/>
