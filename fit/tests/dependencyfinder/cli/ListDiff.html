Checking validation rules

If a row starts with "ensure", it indicates a valid command-line that should work just fine. If a row starts with "reject", it indicates a invalid command-line that contains an invalid set of switches and parameters. The rows for -echo, -help, and -version are special in that they are valid command-lines but do not result in the normal running of the application. This is why I test them using a "check ... false" construct.
